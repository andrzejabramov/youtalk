## 📐 ФОРМУЛЫ

### 1. Диапазон 968-1440px (линейная зависимость)

**Якоря:**

- 967px → margin: 12px, width: 927px
- 1440px → margin: 112px, width: 1200px

**Расчёт коэффициентов:**
Δ экран: 1440 - 967 = 473px
Δ margin: 112 - 12 = 100px → Коэффициент: 473/100 = 4.73
Δ width: 1200 - 927 = 273px → Коэффициент: 473/273 = 1.733

**Формулы:**

```css
margin: calc(12px + (100vw - 967px) / 4.73);
width: calc(927px + (100vw - 967px) / 1.733);

2. Диапазон 600-967px (константа margin)

margin: 12px;
width: calc(100vw - 40px);  /* 40 = 8+8 body + 12+12 container */

3. Диапазон 348-599px (скачок height)

margin: 16px;
width: calc(100vw - 48px);  /* 48 = 8+8 body + 16+16 container */
height: 72px;

4. Диапазон ≤347px (прижатие вправо)

margin-left: auto;
margin-right: 0;
width: 296px;
height: 56px;

💡 ИЗУЧЕННЫЕ МОМЕНТЫ (ОТ АВТОРА)

1. getBoundingClientRect() включает border → использовать фон вместо border для отладки
2. box-sizing: border-box не влияет на замеры → всё равно вычитаем border из расчётов
3. Body margin влияет на расчёт container width → формула: 100vw - (body_margin + container_margin)
4. Скачки на брейкпоинтах допустимы → главное, чтобы внутри диапазона было плавно
5. JS-логгер экономит время → автоматические замеры при resize быстрее ручных
```
