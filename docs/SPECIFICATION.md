## 3. Архитектура и организация кода

### 📁 Зоны ответственности

Проект разделён на логические слои: разметка, стили, логика, данные. Попытка организовать CSS через иерархию папок с `@import` в едином `style.css` была признана нецелесообразной: сервер статических файлов (GitHub Pages) некорректно резолвил пути при импортах, а сборщик периодически терял цепочку каскада.

**Принятое решение:** плоская структура `css/` с **числовыми префиксами** в именах файлов и прямыми `<link>` в HTML. Это гарантирует:

- Детерминированный порядок загрузки (браузер читает файлы строго сверху вниз)
- Корректное отображение исходников в DevTools без потери контекста
- Отсутствие конфликтов с CDN-кэшированием и статическими серверами
- Прозрачную отладку: префикс сразу указывает на зону (`00` → сброс, `10-60` → компоненты, `70-90` → контент и статьи)

### 🧩 Карта структуры

```
├── css/
│   ├── 00-reset.css          # Глобальный сброс, box-sizing
│   ├── 01-variables.css      # Дизайн-токены (цвета, шрифты, отступы)
│   ├── 10-60-*.css           # Базовые компоненты (хедер, навигация, футер, карточки)
│   └── 90-93-*.css           # Контентная часть (сетка статьи, виджеты, адаптивные перескоки)
├── js/
│   ├── include.js            # Асинхронная подгрузка HTML-компонентов
│   ├── article-renderer.js   # Динамический рендер контента из JSON
│   └── data/cards-catalog.json # Каталог карточек
└── components/               # Изолированные HTML-виджеты и блоки
```

## 4. Лейаут: Flex vs Grid

Проведён структурный анализ каждого блока для выбора оптимального инструмента:
| Технология | Где применён | Обоснование |
|------------|--------------|-------------|
| **CSS Grid** | Сетка карточек, правая колонка виджетов (2×2), футер (7 колонок), перескок на `≤974px` | Двумерный контроль над треками, точное выравнивание ячеек, независимость от порядка загрузки DOM |
| **Flexbox** | Навигация, внутренние потоки виджетов, кнопки/текст, облака/декор, мобильные перестроения | Одномерное распределение, естественное обтекание, центрирование, адаптивность `gap` |

**Гибкость:** Для плавного сужения/расширения использованы `calc()`, `clamp()` и формулы вида `215px + (100vw - 975px)/6.93`, что исключает «ступенчатые» скачки и сохраняет пропорции на промежуточных ширинах.

## 5. Интерактивность и логика рендеринга (JS)

# ВАЖНО! (см. скрин сервиса): <screen>

### 🔹 Демо-режим и пагинация

- Клик по эмодзи смайлик на `index.html` → `prompt` с вводом количества карточек (лимит: `21`)
- Ограничение: макс. `9` карточек на страницу пагинации → автоматически создаётся до `3` страниц
- Реализовано для наглядной проверки работы пагинации и сетки без серверной логики

### 🔹 Алгоритм заполнения сетки карточек

Контент берётся из `js/data/cards-catalog.json`. При изменении количества карточек JS динамически формирует DOM и применяет CSS-классы для следующих конфигураций:

| Кол-во | Поведение сетки                                              |
| ------ | ------------------------------------------------------------ |
| `0`    | Заглушка (пустое состояние)                                  |
| `1`    | Одна карточка, центрирована                                  |
| `2`    | 1 ряд, 2 карточки на всю ширину                              |
| `3`    | 1 ряд, 3 карточки                                            |
| `4`    | 2 ряда по 2 карточки                                         |
| `5`    | 1-й ряд: 2 карточки, 2-й ряд: 3 карточки                     |
| `6`    | 2 ряда по 3 карточки                                         |
| `7`    | 2 ряда по 2 + 3-й ряд: 3 карточки                            |
| `8`    | 2 ряда по 2 + 3-й ряд: 3 карточки (адаптив под переполнение) |
| `9`    | 3 ряда по 3 карточки (максимальная плотность)                |

### 🔹 Роутинг и контент

- Кнопка «Читать далее» → гиперссылка на `article.html`
- Статья рендерится динамически: JSON → `DocumentFragment` → `innerHTML` (без блокировки main thread)
- Виджеты загружаются асинхронно через `include.js`, порядок в DOM не влияет на финальную сетку благодаря семантическим классам-обёрткам (`.widget-slot--*`)

## 6. Стилизация и каскад (CSS)

### 🎨 Дизайн-токены

Часто используемые значения вынесены в `01-variables.css`. Это обеспечивает:

- Единую точку правки цветов, шрифтов, отступов, радиусов
- Поддержку темизации без переписывания селекторов
- Валидный `stylelint` и читаемый код

### ⚠️ Использование `!important`

Применён **исключительно как изолятор** в нестандартных случаях, где:

- Базовые стили имели более высокую специфичность, чем требовал адаптивный перескок
- Порядок загрузки файлов создавал конфликт каскада, который невозможно было разрешить без рефакторинга всей ветки
- Требовалось принудительно перебить поведение для нестандартных состояний (например, вписывание виджетов в ячейки Grid `≤974px`)

**Политика:** `!important` документирован, локализован в меди-диапазонах, не используется в базовых стилях. При следующем рефакторинге будет заменён на повышение специфичности или CSS-переменные.

## 7. Деплой и GitHub Pages

### 🌐 Развёртывание

1. `git push` в ветку `main`
2. `Settings → Pages → Source: main / (root) → Save`
3. Сайт доступен по `https://<ник>.github.io/youtalk-blog/`

### ⚡ Особенности платформы

- **Статический хостинг:** отсутствие серверной логики, все данные в JSON/HTML
- **CDN-кэширование:** после пуша изменения могут применяться с задержкой 2–5 минут
- **Решение:** проверка через режим инкогнито или `Ctrl+Shift+R`
- **Относительные пути:** все ссылки и ресурсы используют относительные адреса (`./`, `../`) для корректной работы на Pages без корневых привязок
